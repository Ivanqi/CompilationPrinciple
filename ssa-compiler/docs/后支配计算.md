# 控制流图
```
块0 (ENTRY) → 块1
块1 → 块2, 块3
块2 → 块4
块3 → 块4
块4 → EXIT

```

# 计算得到的支配关系：
## idoms（正向支配）
```
idoms[0] = None     // 入口没有支配者
idoms[1] = Some(0)  // 块1的直接支配者是块0
idoms[2] = Some(1)  // 块2的直接支配者是块1
idoms[3] = Some(1)  // 块3的直接支配者是块1
idoms[4] = Some(1)  // 块4的直接支配者是块1

```

## r_idoms（后支配）
```
r_idoms[0] = Some(1)  // 块0的直接后支配者是块1
r_idoms[1] = Some(4)  // 块1的直接后支配者是块4
r_idoms[2] = Some(4)  // 块2的直接后支配者是块4
r_idoms[3] = Some(4)  // 块3的直接后支配者是块4
r_idoms[4] = None     // 出口没有后支配者

```

# 支配树构建过程
## 正向支配树构建
```
for (i, k) in idoms.into_iter().enumerate() {
    if let Some(_k) = k {
        result.blocks[i].idom = k;        // 设置直接支配者
        result.blocks[_k].idom_of.push(i); // 添加到被支配列表
    }
}

```

### 执行过程：
- `i=1, k=0` → `blocks[1].idom = 0`, `blocks[0].idom_of.push(1)`

- `i=2, k=1` → `blocks[2].idom = 1`, `blocks[1].idom_of.push(2)`

- `i=3, k=1` → `blocks[3].idom = 1`, `blocks[1].idom_of.push(3)`

- `i=4, k=1` → `blocks[4].idom = 1`, `blocks[1].idom_of.push(4)`

### 正向支配树结构
```
块0 (根节点)
  └── 块1
      ├── 块2
      ├── 块3
      └── 块4

```

## 后支配树构建
```
for (i, k) in r_idoms.into_iter().enumerate() {
    if let Some(_k) = k {
        result.blocks[i].r_idom = k;        // 设置直接后支配者
        result.blocks[_k].r_idom_of.push(i); // 添加到被后支配列表
    }
}

```

### 执行过程
- `1`, `blocks[1].r_idom_of.push(0)`

- `i=1, k=4` → `blocks[1].r_idom = 4`, `blocks[4].r_idom_of.push(1)`

- `i=2, k=4` → `blocks[2].r_idom = 4`, `blocks[4].r_idom_of.push(2)`

- `i=3, k=4` → `blocks[3].r_idom = 4`, `blocks[4].r_idom_of.push(3)`

### 后支配树结构
```
块4 (根节点)
  └── 块1
      └── 块0
  ├── 块2
  └── 块3

```
