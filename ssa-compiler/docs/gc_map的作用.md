# gc_map 的作用详解
## gc_map 的定义和填充
```rust
let mut gc_map = vec![usize::MAX; blocks.len()];
// ...
.map(|(new, (old, mut b))| {
    gc_map[old] = new;  // 建立旧索引到新索引的映射
    // ...
})
```

gc_map 结构：
- __索引__：原始块索引（删除前的索引）
- __值__：新块索引（删除后的索引）
- __默认值__：`usize::MAX` 表示该块已被删除


## gc_map 的实际映射示例
假设删除前有5个块：
```
原始索引: [0, 1, 2, 3, 4]
删除块: [2, 4]  // 删除索引为2和4的块
```
删除后:
```
新索引: [0, 1, 3]  // 对应原始索引 [0, 1, 3]

```

gc_map 映射关系
```
gc_map[0] = 0  // 原始0 → 新0
gc_map[1] = 1  // 原始1 → 新1  
gc_map[2] = usize::MAX  // 原始2 → 已删除
gc_map[3] = 2  // 原始3 → 新2
gc_map[4] = usize::MAX  // 原始4 → 已删除

```

## 重映射过程示例
### 删除前的控制流关系
```
块0: preds=[], children=[1, 2]
块1: preds=[0], children=[3]
块2: preds=[0], children=[4]  // 将被删除
块3: preds=[1], children=[4]  // 将被重映射
块4: preds=[2, 3], children=[]  // 将被删除

```
### 重映射后的控制流关系
```
块0: preds=[], children=[1]  // 删除了指向2的边
块1: preds=[0], children=[2]  // 3→2重映射
块2: preds=[1], children=[]   // 4→删除

```
