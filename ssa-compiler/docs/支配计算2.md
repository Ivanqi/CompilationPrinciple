# 1. 遍历所有基本块
```rust
for (i, block) in self.blocks.iter().enumerate() {
```
- 遍历控制流图中的每个基本块
- `i` 是当前块的索引
- `block` 是当前基本块的引用

# 2. 计算新的支配者集合
```rust
let mut new: HashSet<_> = if let Some(fst) = selector(block).first() {
    selector(block)
        .iter()
        .map(|n| &doms[*n])
        .fold(doms[*fst].clone(), |acc, next| {
            acc.intersection(next).cloned().collect()
        })
} else {
    HashSet::new()
};

```
核心逻辑:
- __选择器函数__：获取前驱（支配计算）或后继（后支配计算）
- __交集运算__：计算所有前驱支配者集合的交集
- __空集处理__：如果没有前驱，返回空集

# 3. 添加当前块到支配者集合
```rust
new.insert(i);
```
- 每个块都支配自己
- 这是支配关系的自反性

# 4. 更新支配者集合
```rust
if new != doms[i] {
    changed = true;
    doms[i] = new;
}
```
- 如果新的支配者集合与旧的不同，标记变化
- 更新 doms[i] 为新的支配者集合

# 与 doms 变量关联
## `doms` 变量的作用
```rust
let mut doms = vec![HashSet::from_iter(0..self.blocks.len()); self.blocks.len()];
```
- __结构__：`Vec<HashSet<usize>>`，每个元素是一个块的支配者集合
- __初始值__：每个块支配所有块（除了根节点）
- __最终值__：每个块的精确支配者集合


## 数据流关系
### 支配者计算的数据流方程：
```rust
doms[i] = {i} ∪ (∩_{p ∈ preds(i)} doms[p])
```

### 代码对应关系
```rust
// {i} ∪ (∩_{p ∈ preds(i)} doms[p])
new.insert(i);  // {i} ∪
selector(block).iter().map(|n| &doms[*n]).fold(...)  // ∩_{p ∈ preds(i)} doms[p]
```

## 具体示例说明:
### 控制流图:
```
块0 → 块1
块1 → 块2, 块3
块2 → 块4
块3 → 块4
```

### 迭代过程
#### 计算块4的支配者
```
// 前驱：{块2, 块3}
// doms[2] = {0,1,2}
// doms[3] = {0,1,3}
// 交集：{0,1,2} ∩ {0,1,3} = {0,1}
// 加上自己：{0,1,4}
new = {0,1,4}
```
- __读取__：从 `doms[2]` 和 `doms[3]` 读取前驱的支配者集合
- __计算__：计算交集并添加当前块
- __写入__：将结果写入 `doms[4]`
