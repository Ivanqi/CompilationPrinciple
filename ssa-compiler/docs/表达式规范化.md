# 代码核心逻辑
```rust
fn canon(self) -> Self {
    match self {
        Expression::Plus(v1, v2) => {
            Expression::Plus(std::cmp::min(v1, v2), std::cmp::max(v1, v2))
        }
        Expression::Mult(v1, v2) => {
            Expression::Mult(std::cmp::min(v1, v2), std::cmp::max(v1, v2))
        }
        Expression::And(v1, v2) => {
            Expression::And(std::cmp::min(v1, v2), std::cmp::max(v1, v2))
        }
        Expression::Or(v1, v2) => {
            Expression::Or(std::cmp::min(v1, v2), std::cmp::max(v1, v2))
        }
        Expression::Xor(v1, v2) => {
            Expression::Xor(std::cmp::min(v1, v2), std::cmp::max(v1, v2))
        }
        e => e,
    }
}
```

# 算法原理详解
## 1. 交换律保证
```
// 规范化确保交换律成立：
// Plus(a, b) ≡ Plus(b, a)
// Mult(a, b) ≡ Mult(b, a)
// And(a, b) ≡ And(b, a)
// Or(a, b) ≡ Or(b, a)
// Xor(a, b) ≡ Xor(b, a)

```

## 2. 值编号一致性
```rust
// 规范化前：
let expr1 = Expression::Plus(10, 5);  // 值编号：15
let expr2 = Expression::Plus(5, 10);  // 值编号：16（不同！）

// 规范化后：
let canon1 = expr1.canon();  // Plus(5, 10) 值编号：15
let canon2 = expr2.canon();  // Plus(5, 10) 值编号：15（相同！）
```

### 3. 哈希一致性
```
// 规范化确保相同的表达式获得相同的哈希值
// 从而在HashMap中获得相同的键
```

# 技术意义
## 1. 优化效果
```

// 规范化后：只存储一个规范化形式
```
